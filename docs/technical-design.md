## 二、核心模块技术设计

### 2.1 模块一：2D→3D 语义门控（Semantic Gate）

**目标**：避免对每帧都触发计算密集的3D几何网络，仅在场景发生显著语义变化时激活。

**设计方案**：

| 要素 | 说明 |
|------|------|
| 输入 | 当前帧 $f_t$ 与历史帧 $f_{t-k}$ 的 2D CLS token 特征 $z_t, z_{t-k} \in \mathbb{R}^d$ |
| 差异度量 | 余弦距离：$\delta_t = 1 - \frac{z_t \cdot z_{t-k}}{\|z_t\|\|z_{t-k}\|}$ |
| 门控决策 | $g_t = \mathbf{1}[\delta_t > \tau]$，阈值 $\tau$ 通过验证集网格搜索确定 |
| 输出 | $g_t=1$ 时触发 VGGT-1B 推理；$g_t=0$ 时复用上一帧3D缓存 |
| 参数量 | **零额外可训练参数**（纯规则判断） |

**关键超参**：
- 历史窗口步长 $k \in \{1, 2, 3\}$
- 门控阈值 $\tau \in [0.05, 0.30]$（步长0.05网格搜索）

---

### 2.2 模块二：3D→2D 几何引导压缩（Geometry-guided Zip）

**目标**：利用3D几何先验生成无监督重要性图，对 ViT patch token 进行动态裁剪，去除与空间任务无关的冗余 token。

**设计方案**：

**Step 1 — 重要性图生成**

$$
S_i = \alpha \cdot \sigma^2(\mathbf{D}_i) + \beta \cdot H(\mathbf{P}_i)
$$

其中：
- $\sigma^2(\mathbf{D}_i)$：第 $i$ 个 patch 区域内深度图的局部方差（反映几何复杂度）
- $H(\mathbf{P}_i)$：位姿不确定性熵（由 VGGT 输出的置信度图导出）
- $\alpha, \beta$：平衡系数，默认 $\alpha=0.6, \beta=0.4$

**Step 2 — 无参 Top-K 裁剪**

$$
\mathcal{T}_{keep} = \text{TopK}(\{S_i\}_{i=1}^{N},\ k = \lfloor r \cdot N \rfloor)
$$

保留率 $r \in \{0.25, 0.50, 0.75\}$，默认 $r=0.50$。

**Step 3 — 位置编码修正**

保留原始绝对位置编码，确保 LLM 感知 patch 空间位置不变。

| 要素 | 说明 |
|------|------|
| 参数量 | **零额外可训练参数** |
| 监督信号 | 无需3D真值标注（自监督） |
| 预期压缩率 | token 数量降至原始的 25%～50% |

---

### 2.3 模块三：增量 KV 缓存（Incremental KV Cache）

**目标**：实现真正的逐帧流式推理，推理复杂度从 $O(T)$ 降至 $O(w)$（$w$ 为滑动窗口大小）。

**设计方案**：

| 组件 | 说明 |
|------|------|
| 3D 隐状态队列 | 维护最近 $w$ 帧的 VGGT 特征，FIFO 驱逐策略 |
| 2D 特征缓存 | 存储压缩后的 patch token 特征，按帧时间戳索引 |
| KV-Cache 管理 | 新帧 token 追加写入；超出窗口的旧 token 按重要性分数驱逐 |
| 窗口大小 $w$ | 默认 $w=8$，消融实验覆盖 $\{4, 8, 16, 32\}$ |

**流式推理伪代码**：

```python
cache = IncrementalKVCache(window=w)
for frame_t in video_stream:
    feat_2d = vit_encoder(frame_t)          # 2D 特征提取
    if gate(feat_2d, cache.last_feat):      # 2D→3D 门控判断
        feat_3d = vggt(frame_t)             # 触发 3D 推理
        cache.update_3d(feat_3d)
    else:
        feat_3d = cache.get_last_3d()       # 复用缓存
    tokens = geo_zip(feat_2d, feat_3d)      # 几何引导压缩
    cache.append(tokens)                    # 增量写入 KV
    if query_arrived:
        answer = llm_decode(cache)          # 流式解码
```

## 七、风险评估与应对策略

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| VGGT-1B 推理速度不达预期 | 中 | 高 | 改用 Depth-Anything-V2 替代深度提取，降低3D模块开销 |
| SPAR-7M 精度下降超过2% | 中 | 高 | 调高保留率 $r$（0.75）或降低门控阈值 $\tau$ |
| RTX 3060 显存不足 | 低 | 中 | 启用 gradient checkpointing；推理时使用 FP16 |
| 基线复现结果与论文不符 | 低 | 中 | 使用官方代码仓库，记录环境差异 |
| 流式推理延迟不稳定 | 低 | 低 | 固定随机种子，多次运行取均值 |
